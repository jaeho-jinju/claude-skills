# HTML 발표자료 생성기

다크 테마·그라디언트 스타일의 HTML 발표자료를 생성한다.
단일 `.html` 파일로 출력되며 브라우저에서 바로 발표 가능하다.

## Step 1: 정보 수집

다음 세 가지가 없으면 먼저 물어볼 것.

1. **발표 주제** — 무엇에 대한 발표인가?
2. **주요 내용 / 메시지** — 전달할 핵심 포인트는 무엇인가?
3. **슬라이드 수** (선택) — 없으면 내용 분량에 맞게 자동 결정

대상 청중·시간은 선택 정보 (알면 톤/분량 조정에 활용).

## Step 2: 슬라이드 구성 기획

내용에 따라 적합한 레이아웃 타입을 선택한다.

### 레이아웃 타입 선택 기준

| 타입 | 선택 기준 |
|------|----------|
| `title` | 첫/마지막 슬라이드 |
| `section-title` | 섹션 전환 — SECTION N 배지 + 대형 그라디언트 제목 + 부제 |
| `cards-3` | 3개 지표·개념·통계 강조 (숫자나 이모지 사용) |
| `cards-2` | 2개 개념 나란히 비교·설명 |
| `cards-with-insight` | 2~3개 카드 + 하단 인사이트 박스 조합 (핵심 요약이 필요할 때) |
| `before-after` | BEFORE/AFTER 대조 — 붉은 패널(문제) vs 청록 패널(해결) + 중앙 VS 배지 |
| `comparison` | A vs B 다차원 비교 (행별로 항목 비교) |
| `comparison-table` | 3~4개 항목 행별 비교 + 각 셀에 배지·진행 바 포함 (상세 스펙 비교) |
| `quality-tiers` | 나쁨/보통/좋음 등급 표시 — X(빨강)·△(주황)·✓(초록) 아이콘 + 예시 인용 |
| `timeline` | 단계적 흐름·역사·진행 (3~4단계) |
| `numbered-list` | 3~5개 항목 설명 (순서/단계, 특정 항목 강조 가능) |
| `definition` | 핵심 개념 정의 + 서포팅 카드 |
| `stages` | 단계 진행 흐름 (화살표 연결, 특정 단계 강조) |
| `node-diagram` | 원형 노드 연결 다이어그램 — 메인 노드 + 연결된 서브 노드 관계 표현 |
| `code-annotated` | 코드·설정 파일 블록 + 우측/하단 어노테이션 카드 |
| `mixed` | 상단 카드/단계 + 하단 바 차트·플로우·체크리스트 조합 |

### 슬라이드 구조 원칙
- 첫 슬라이드: `title` 타입
- 마지막 슬라이드: `title` 타입 (결론 또는 Q&A)
- 섹션 구분: `section-title` 타입 사용
- 본론: 내용에 맞는 타입 혼합 (같은 타입 연속 3회 이상 지양)

## Step 3: 슬라이드 구성 채팅에 출력

HTML 생성 전 아래 형식으로 먼저 보여준다.

```
## 슬라이드 구성 (총 N장)

### 슬라이드 1: [제목] — [타입]
- 핵심: ...

### 슬라이드 2: [제목] — [타입]
- 핵심: ...
```

## Step 4: HTML 파일 생성

`assets/base_template.html`을 먼저 읽어서 CSS·JS 구조를 파악한 뒤,
그 스타일 시스템을 그대로 사용해 실제 내용으로 채운 HTML 파일을 생성한다.

### 생성 절차

1. **Read** `assets/base_template.html` — CSS 변수·클래스 확인
2. 템플릿의 CSS + JS를 그대로 복사
3. `<div class="presentation">` 안의 예시 슬라이드를 실제 내용으로 교체
4. Write 툴로 최종 파일 저장

### 파일명 규칙
`<주제_영문요약>_slides.html` — 현재 작업 디렉토리에 저장
예: `ai_engineering_slides.html`, `bodoc_architecture_slides.html`

## Step 4-1: 타이포그래피 & 인터랙션 (v2.0+)

### 텍스트 크기 표준 (가독성 최적화)

| 요소 | 크기 | 용도 | 주의사항 |
|------|------|------|---------|
| **타이틀 (title-main)** | 72px | 첫 슬라이드, 최상위 제목 | 강렬하고 명확함 |
| **슬라이드 제목 (.slide-title)** | 52px | 각 슬라이드 제목 | — |
| **서브타이틀 (.slide-subtitle)** | **21px** ✨ | 제목 아래 설명 | v1.0에서 17px → 21px 개선 (가독성↑) |
| **타이틀 서브 (.title-sub)** | 22px | 타이틀 슬라이드 부제 | — |
| **카드 설명 (.stat-desc)** | **15px** ✨ | 카드 내 설명 텍스트 | v1.0에서 14px → 15px 개선 |
| **리스트 제목 (.num-title)** | 16px | 번호 목록 항목 제목 | — |
| **리스트 설명 (.num-desc)** | 14px | 번호 목록 항목 설명 | — |
| **타임라인 설명 (.tl-desc)** | **14px** ✨ | 타임라인 단계 설명 | v1.0에서 13px → 14px 개선 |
| **캡션 (.caption)** | 15px | 하단 주석 | — |

**📌 개선 이유:** 프레젠테이션 거리에서 뒤에 앉은 청중도 충분히 읽을 수 있도록 조정.

### 호버 효과 시스템 (v2.0+)

모든 인터랙티브 요소에 부드러운 호버 효과 적용. **화면 흔들림 없음** (transform 대신 색상 + 글로우만 사용).

#### 카드 호버
```css
.card:hover {
  background: rgba(18, 18, 42, 0.9);  /* 배경 진하게 */
  box-shadow: 0 8px 28px rgba(168, 85, 247, 0.4);  /* 보라색 글로우 */
}
```
- 마우스를 카드 위에 올리면 배경색이 진해지고 글로우 효과
- UI 안정성: 레이아웃 변화 없음 (색상만 변경)

#### 제목 호버
```css
.slide-title:hover {
  text-shadow: 0 0 20px rgba(168, 85, 247, 0.5);  /* 텍스트 글로우 */
}
```
- 제목에 마우스를 올리면 텍스트가 발광

#### 번호 항목 호버
```css
.num-item:hover {
  background: rgba(168, 85, 247, 0.18);  /* 배경 강조 */
  box-shadow: 0 6px 20px rgba(168, 85, 247, 0.25);  /* 은은한 글로우 */
}
```
- 리스트 항목에 마우스를 올리면 강조됨

#### 네비게이션 버튼 호버 (JavaScript)
```javascript
btn.addEventListener('mouseenter', () => {
  btn.style.backgroundColor = 'rgba(168, 85, 247, 0.6)';  /* 보라색 */
  btn.style.transform = 'scale(1.15)';  /* 15% 확대 */
  btn.style.boxShadow = '0 0 24px rgba(168, 85, 247, 0.8)';  /* 강한 글로우 */
});
```
- 좌우 화살표 버튼에 마우스를 올리면 보라색으로 밝아지고 확대
- 버튼은 작은 요소이므로 scale 효과 사용 가능

#### 진행 점 호버 (JavaScript)
```javascript
dot.addEventListener('mouseenter', () => {
  dot.style.backgroundColor = 'rgba(168, 85, 247, 0.8)';  /* 보라색 */
  dot.style.transform = 'scale(1.5)';  /* 50% 확대 */
  dot.style.boxShadow = '0 0 16px rgba(168, 85, 247, 0.6)';  /* 글로우 */
});
```
- 하단 진행 점에 마우스를 올리면 보라색으로 강조되고 확대

**⚠️ 주의:** 슬라이드 내용(카드, 리스트)의 호버는 색상만 사용하여 레이아웃 변화 없음. 네비게이션 요소(버튼, 점)의 호버만 scale 효과 사용.

## 디자인 시스템

### 슬라이드 제목 스타일

- 흰색(`.white`): 단호한 선언형 ("A는 B다", "3명·5개월·0줄")
- 그라디언트(`.gradient`): 개념명·섹션 제목 ("AI 엔지니어링", "CLAUDE.md vs Skills")
- 웜 그라디언트(`.gradient-warm`): 경고·전환 포인트, 품질/중요도 강조

### 색상 팔레트

| 색상 | 주요 용도 |
|------|----------|
| 보라 (purple) | 첫 번째 카드, CLAUDE.md 계열, 핵심 개념 |
| 청록 (cyan) | 두 번째 카드, Skills 계열, 긍정·경량 |
| 주황 (orange) | 세 번째 카드, Sub-Agent 계열, 강조·경고 |
| 초록 (green) | 완료·정답·AFTER 패널, 성공 지표 |
| 빨강 (red) | 문제·BEFORE 패널, 나쁜 사례 |
| 황금 (gold) | 추천 강조, 배지 하이라이트 |

**카드 3개:** purple → cyan → orange/green
**카드 2개:** 대비되는 색 (orange + cyan, red + green)
**BEFORE/AFTER:** 빨강/마룬 배경(BEFORE) + 청록/다크그린 배경(AFTER)

### 배지 & 레이블

| 배지 | 용도 |
|------|------|
| `STAGE N` | 단계 카드 상단 — 처리 단계 구분 |
| `SECTION N` | 섹션 전환 슬라이드 상단 |
| `방법 N` / `방법 N — 추천` | 선택지 카드 구분, 추천 표시 |
| `BEFORE` / `AFTER` | 전후 비교 패널 상단 |
| `✓ Lightweight` / `⚡ On-Demand` | 특성·기능 태그 |
| `나쁨` / `보통` / `좋음` | 품질 등급 레이블 (빨강·주황·초록) |

### 진행 바 (Progress Bar)
수준·비율·비교를 시각화할 때 사용.
- 카드 내부에 색상 진행 바로 상대적 크기 표현
- BEFORE는 짧게, AFTER는 길게 — 개선 효과 시각화
- 비교 테이블 각 행에 purple(좌) / cyan(우) 진행 바로 대비

### 인사이트 박스 (Insight Box)
슬라이드 하단 핵심 요약 영역. `cards-with-insight`, `comparison-table` 하단에 배치.
- 어두운 배경, `→` bullet으로 시작
- 레이블: "핵심 인사이트", "정리하면", "핵심 포인트"
- 왼쪽 컬러 보더로 강조 (보라·주황·청록)

### 캡션 바 (Caption Bar)
슬라이드 최하단 핵심 메시지 1줄 요약.
- `⚠️` + "핵심 포인트" 레이블 + 굵은 메시지
- 주황 왼쪽 보더

### 코드 에디터 블록
코드·설정 파일·구조를 보여줄 때 사용.
- 어두운 배경 + 상단 파일명 탭
- 구문 강조: YAML 키(청록), 값(흰색/주황), 주석(회색)
- 우측 또는 하단에 어노테이션 카드 배치 (`code-annotated` 타입)

### 플로우 다이어그램
프로세스 흐름을 수평 화살표로 표현. `mixed` 타입 하단에 배치.
- 박스 → 화살표 → 박스 구조
- 최종 결과물 박스는 주황/붉은 배경으로 강조

### 노드 다이어그램 (`node-diagram`)
계층적 관계·네트워크를 원형 노드로 표현.
- 메인 노드 (중앙, 큰 원) → 서브 노드 (주변, 작은 원)
- 각 노드에 이모지 + 레이블
- 연결선 색상으로 관계 유형 구분

## 내용 작성 지침

- 슬라이드 제목은 10자 이내로 간결하게
- 카드 설명은 2~3줄 이내
- 리스트 항목은 4개 이하 유지
- 핵심 키워드는 `<span class="hl-purple">` 등으로 인라인 강조
- 캡션(하단 주석)으로 슬라이드의 핵심 메시지 1줄 요약

### 이모지 사용
아이콘 대신 이모지를 적극 활용한다.

| 카테고리 | 이모지 |
|---------|--------|
| 통계/수치 | 🎯 📊 ⚡ 🚀 |
| 사람/팀 | 👤 👥 🧑‍💻 🤖 |
| 시간/일정 | 📅 ⏱️ |
| 코드/기술 | 💻 🔧 ⚙️ 📋 |
| 위험/경고 | ⚠️ 🚧 🔒 |
| 성공/완료 | ✅ 🏆 ✨ |
| 탐색/검색 | 🔍 🧭 |

## Step 5: 파일 열기 + 검증

파일 생성 후:
1. `open <파일경로>` 명령으로 브라우저에서 바로 열기
2. 검증 체크리스트 출력

```
## 검증 체크리스트

- [x/] HTML 파일이 생성되었는가?
- [x/] 첫 슬라이드가 타이틀 슬라이드인가?
- [x/] 모든 슬라이드에 제목이 있는가?
- [x/] 키보드 ← → 네비게이션 JS가 포함되었는가?
- [x/] 다크 테마 + 그라디언트 스타일이 적용되었는가?
- [x/] 슬라이드 번호/도트 네비게이션이 표시되는가?

결과: N/6 통과
파일 위치: /path/to/file.html
```
